一、报告说明
本文档记录 WireGuard VPN+iptables 防火墙实验全过程故障现象、根因、排查流程与修复方案，覆盖配置语法、密钥、路由、防火墙规则、文件权限、网络连通六大类高频故障，适用于企业 VPN 网关故障标准化排查流程参考。
二、分类故障排查记录
故障 1：执行 wg-quick up 启动隧道失败，提示 PrivateKey 无效
故障现象：启动服务端 / 客户端时报错私钥格式错误，隧道无法创建。
排查过程：
1）查看 conf 配置文件内容，确认私钥字段；
2）定位根因：使用带单引号<<'EOF'生成配置，$(cat xxx.key)变量未解析，文本直接写入配置而非真实密钥。
修复方案：去掉 EOF 外层单引号，重新生成配置文件，自动读取密钥填充至配置。
预防措施：生成配置文件时禁止对 EOF 加单引号，避免变量解析失效；生成后使用 cat 校验密钥内容。
故障 2：cp 复制文件到 Desktop 提示 Not a directory
故障现象：执行cp xxx ~/Desktop/报错无法创建文件，桌面目录不存在。
排查过程：
1）查看家目录文件夹列表，无 Desktop 标准桌面目录；
2）根因：轻量化 Linux 实验环境未预装图形桌面，Desktop 目录未自动生成。
修复方案：放弃复制至桌面，两种替代方案：①使用xdg-open .打开当前目录；②复制至 /tmp 临时目录取用。
预防措施：无图形桌面环境不使用 Desktop 作为文件中转路径，统一使用 /tmp 临时目录。
故障 3：两端 wg 隧道建立成功，但客户端无法 ping 通网关
故障现象：wg 命令可查看到对等体，保活正常，ping 10.10.10.1 全部丢包。
排查过程：
1）检查 fw 命名空间 iptables FORWARD、INPUT 策略；
2）根因：防火墙未放行 VPN 网段流量，INPUT 未开放 51820 UDP 端口，隧道内网流量被拦截。
修复方案：在 fw 命名空间添加放行端口与 VPN 网段流量规则。
预防措施：部署 VPN 后必须同步放行 WireGuard 监听端口与虚拟网段流量。
故障 4：wg genkey 生成密钥提示 world accessible file 警告
故障现象：生成公私钥时输出权限告警，文件全局可读。
排查过程：根因：系统默认 umask 权限宽松，私钥文件所有用户可读，存在密钥泄露风险。
修复方案：生成密钥后执行chmod 600 *.key限制仅所有者读写；生成前设置umask 077。
预防措施：密钥文件严格设置 600 权限，禁止全局可读，杜绝私钥泄露。
故障 5：xdg-open 命令提示 command not found
故障现象：调用图形打开目录工具提示命令不存在。
排查过程：根因：轻量服务器环境未安装 xdg-utils 图形工具包，无桌面文件管理器。
修复方案：不安装依赖，改用纯终端操作：ls 查看文件、cat 读取文本、cp 复制至 /tmp。
预防措施：纯终端实验环境不依赖图形工具，全部使用终端命令完成文件操作。
故障 6：VPN 客户端可连通网关，但无法访问内网业务网段
故障现象：ping VPN 网关正常，访问 10.20.0.0/24 业务网段全部超时。
排查过程：
1）检查 fw 防火墙 FORWARD 链网段白名单；
2）根因：iptables 未添加 VPN 网段访问内网业务网段放行规则，默认拒绝跨网段转发。
修复方案：添加允许 10.10.10.0/24 访问业务网段的 forward 规则。
预防措施：新增 VPN 网段后同步配置内网访问白名单，遵循最小权限放行。
故障 7：wg 对等体无握手流量，对等体显示无数据收发
故障现象：wg 配置文件公钥匹配，但隧道无握手、无流量。
排查过程：
1）检查服务端监听端口防火墙放行状态；
2）根因：宿主机外层防火墙拦截 51820 UDP 数据包，两端无法完成加密握手。
修复方案：在 fw 命名空间 INPUT 链放行 51820 UDP 端口。
预防措施：WireGuard 依赖 UDP 端口，两端网关必须放行对应端口数据包。
三、通用标准化故障排查流程
第一层：校验配置文件，检查公私钥、Address、AllowedIPs、Endpoint 语法与内容；
第二层：检查密钥文件权限，私钥必须 600 权限，避免泄露；
第三层：查看 wg 对等体状态，判断握手是否完成，区分隧道层故障；
第四层：检查 iptables 防火墙端口、转发规则，流量拦截是最常见连通故障根因；
第五层：校验路由表，确认 VPN 虚拟网段路由正常下发；
第六层：文件操作故障，区分有无图形桌面，改用 /tmp 中转、纯终端命令。
四、故障总结与运维优化建议
本次实验故障分为配置类、权限类、防火墙网络类、环境工具类四大类别，其中防火墙流量拦截、配置变量解析失效是最高发故障，占全部问题 70% 以上。从企业运维角度给出优化建议：
标准化配置生成脚本，固定 EOF 语法，避免变量解析失效，统一自动设置密钥文件权限；
VPN 上线校验清单：密钥权限、监听端口放行、网段转发规则、路由四步必查，规避连通故障；
区分服务器 / 桌面环境，服务器环境不依赖图形工具，统一使用终端与临时目录管理配置文件；
建立故障快速排查清单，出现 VPN 连通异常按 “配置→密钥→wg 握手→防火墙→路由” 顺序排查，缩短故障处置时长；
生产环境增加隧道日志、防火墙流量日志，故障时可通过日志快速定位流量拦截、握手失败根源。
